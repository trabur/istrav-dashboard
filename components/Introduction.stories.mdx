# FARMERLESS: Frontend web components combined with headless backend.

### Powered by ISTRAV.

What makes these web components different from other components? First of all, they are highly opinionated; we make assumptions about which technologies to use. You may like some of the decisions that we have made or you may not like one particular thing or another. Such as our decision to use material design as our CSS library. Or perhaps Svelte as our frontend framework. In the future we may branch out and be party agnostic. However, our focus right now is around getting everything on point before we move forward. Because, second of all, the challenge that we face is not just having web components but a having them and being able to hydrate them with data. In short, we need web components that are dynamic. Note: later we explain what blocks and wireframes are. Then finally, we need our web components to support community based interaction on a Command Query Responsibility Segregation (CQRS) level with event sourcing. Like having users with roles that have permissions or also known as identification access management (IAM). Now setting up a profile page where only users can edit their own account becomes possible. Many call this, "feature flagging". In short, we already made many of the tough opninionated decisions for you, web components may be hydrated with dynamic data, and lastly web components are event source driven so functionality such as feature flagging exists.
